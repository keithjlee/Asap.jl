# Asap v1.0.0

A ground-up rewrite of the analysis core. v1.0 unifies trusses and frames in one model, makes connection stiffness first-class and differentiable, recovers exact internal force and displacement fields along members, and exposes a pure functional analysis path that AD engines (Zygote, Mooncake, Enzyme) traverse natively. The entire legacy v0.2.x numerical behavior is pinned by a characterization test suite — where v1.0 deviates, it is deliberate and documented below.

## New features

**One model for everything**
- `TrussElement` and `FrameElement` mix freely in a single `Model` — the duplicated truss/frame pipelines are gone.
- Automatic DOF activity (free / fixed / **inactive**): rotational DOFs that nothing stiffens (truss nodes, fully released ends) are excluded from the solve by construction. No more singular stiffness matrices from hinge/torsion modes, no manual bookkeeping.
- Parametric types throughout (`Model{T}`, `Material{T}`, …): dual numbers and arbitrary precision flow through the core.

**Connections as data**
- End releases are now **end springs** (`EndSprings`/`EndConditions`, Monforton–Wu fixity factors). The classical releases (`:fixedfixed`, `:fixedfree`, `:freefixed`, `:freefree`, `:joist`) are exact limits; any finite stiffness is a semi-rigid connection — differentiable, and usable as a design variable in AsapOptim (`JointVariable`).
- `VariableElement`: one member with a varying cross-section as a chain of prismatic segments via internal DOFs — replaces `BridgeElement` and the fragile model-shattering it required.
- `NodalSpring`: elastic (spring) supports per global DOF.
- `RigiditySection`: stores `EA`/`EIx`/`EIy`/`GJ`/`ρA` directly — the natural type for cracked/effective concrete stiffness and homogenized members.

**Loads**
- `DistributedLoad`: piecewise-linear intensity as the canonical element load; `LineLoad` and new `TrapezoidLoad` lower to it. New `PointMoment`. New `SelfWeight`, which varies with each segment's `ρA` on variable elements (and replaces the legacy `GravityLoad`, which never worked).
- One Gauss-quadrature fixed-end-force engine, exact for all supported loads, with generic condensation for any end condition — replaces the per-release formula catalog.

**Results and member fields**
- Results live on `LinearResults`, queried through accessors (`displacement(res, node)`, `reaction(res, node)`, `element_forces(res, el)`, `axial_force(res, el)`) — nothing is mutated onto nodes/elements anymore.
- Exact internal force **and displaced shape** recovery along any member, for any end condition: scalar zero-allocation evaluators (`moment_z(model, el, t)`, `shear_y`, …), dense `InternalForces` sampling with true jumps at point loads, and `local_displacements` for exact elastic curves.
- Load cases, `LoadCombination`s, and station-wise `envelope`s — solved as multi-RHS from a single factorization.

**Differentiable analysis**
- Pure functional path (`extract_state` → `ModelState` → `solve`) sharing one set of element kernels with the in-place solver; `ChainRulesCore` rules ship as a package extension.
- Works out of the box with **Zygote, Mooncake, and Enzyme** (via DifferentiationInterface); Enzyme gradients on a 512-variable spaceframe outperform the legacy hand-differentiated AsapOptim stack.

**Performance**
- Frozen sparsity + zero-allocation assembly + cached/reused factorization: vs v0.2.2, solves ~1.2× faster, assembly ~1.5× faster, differentiable forward pass ~2× faster on a representative benchmark spaceframe.

**Batteries included**
- The parametric structure generators moved into Asap from AsapToolkit: `Warren2D`, `Pratt2D`, `BakerTruss`, `TrussFrame`, `SpaceFrame` (incl. arbitrary surface callables), `SpaceFrameBeam`, `Frame`, `GridFrame`, `GridNetwork`, and the ground-structure family — plus `Geo` plot-prep, `ElementDisplacements`, and `to_network` (model ↔ FDM translation).
- Every exported function and type carries a full docstring with dimensions in bracket notation, and every type prints an engineer-readable REPL summary.
- 2,600+ tests: textbook solutions, pinned legacy numerics, published-paper structures and gradients, closed-form beam formulas, AD-vs-finite-difference checks, and zero-allocation guarantees.

## Breaking changes

This is a clean break from v0.2.x. The essentials:

| v0.2.x | v1.0 |
|---|---|
| `TrussModel(nodes, els, loads)` | `Model(nodes, els, loads)` — one type for everything |
| `TrussNode(pos, fixity)` | `Node(pos, fixity)` — always 6 DOF slots; unused rotations go inactive automatically |
| `TrussSection(A, E)` | `Section(mat, A)` |
| `Element(n1, n2, sec, release)` | `FrameElement(n1, n2, sec; release = :fixedfree)` or an explicit `EndConditions` |
| `BridgeElement(...)` | `VariableElement(n1, n2, sections, breaks)` |
| `GravityLoad(el, factor)` | `SelfWeight(el, gvec)` |
| `node.displacement`, `node.reaction` | `displacement(res, node)`, `reaction(res, node)` with `res = model.results` |
| `element.forces` | `element_forces(res, el)`; axial via `axial_force(res, el)` (tension-positive) |
| `element.K` / `.R` / `.Q` / `.LCS` / `.length` | `stiffness(el, x1, x2)` / `local_frame(el)` / `length(el)` |
| AsapToolkit `InternalForces(el, model)` | `InternalForces(model, el)` in Asap itself — **axis-correct names**: old `.P` → `N`, old `.My` → `Mz`, old `.Mz` → `My` |

Known v0.2.x bugs are fixed rather than reproduced — notably `GravityLoad` fixed-end forces (never functional) and the axial component of an off-center `PointLoad` (was split 50/50; now distributed by the lever rule).

**Dependencies/floor**: Julia 1.10+; StaticArrays is now a hard dependency; ChainRulesCore, Enzyme, and Mooncake are weak dependencies (package extensions).

## Ecosystem

Updated in lockstep: **AsapOptim v1.0** (rewritten on the pure differentiable path — its reverse-engineered adjoint layer is gone), **AsapToolkit v1.0** (now the AISC section database, polygonal section geometry, and Grasshopper/JSON IO; generators and diagrams live in Asap), **AsapHarmonics**, **DemandTransport2**.
