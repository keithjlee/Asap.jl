# Asap v1.1.1

Bug-fix release: restores Enzyme reverse-mode differentiation, broken in
v1.1.0.

## Fixed

- **Enzyme reverse mode failed to compile** (`Enzyme Internal Error:
  Illegal calling convention fixup`) on anything passing through
  `solve_free`. The v1.1.0 solver-seam refactor routed the `solve_free`
  rrule through the `FactorizationCache` wrapper, whose deliberately
  untyped fields put dynamic dispatch inside the rule that
  `Enzyme.@import_rrule` compiles. The imported (2-arg) rule now uses the
  raw concretely-typed factorization again, with a flat literal-tuple
  pullback; the 3-arg (solver-seam) rule keeps the wrapper, which
  ChainRules-based engines handle fine. Zygote/Mooncake/ForwardDiff were
  unaffected throughout; numerics unchanged (suite: 2718).

If you use Enzyme with Asap v1.1.0, upgrade.
